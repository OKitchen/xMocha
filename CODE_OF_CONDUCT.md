# xMocha Code of Conduct

xMocha is an open-source project for decision simulation, agent worlds, and replayable branching futures. We want the community to be thoughtful, practical, and safe for people discussing both code and sensitive decision-support use cases.

## Expected Behavior

- Be respectful, direct, and constructive.
- Keep feedback focused on code, docs, behavior, product clarity, safety, or reproducible evidence.
- Assume good intent, and adjust quickly when someone says an interaction is unhelpful or harmful.
- Protect private data. Do not post API keys, database URLs, private user dilemmas, contact details, raw uploads, hidden prompts, or production logs.
- Be careful with claims about xMocha. Do not present it as future prediction, medical/legal/financial advice, therapy, or an automatic decision-maker.

## Unacceptable Behavior

- Harassment, threats, insults, discrimination, or unwanted personal attention.
- Publishing private information without permission.
- Posting exploit details, secrets, private user data, or sensitive logs in public issues or pull requests.
- Repeatedly derailing technical discussions after maintainers ask to refocus.
- Submitting copyrighted, private, or unclear-rights data as examples, tests, worlds, or training material.

## Reporting

If you see a conduct issue, contact the maintainers privately. Use the subject:

```text
[Conduct] xMocha community report
```

For security or privacy vulnerabilities, follow `SECURITY.md` instead of opening a public issue.

Please include enough context for maintainers to understand what happened, while redacting secrets, private user data, and sensitive logs.

## Enforcement

Maintainers may remove comments, close issues, reject pull requests, request edits, temporarily limit participation, or block users when needed to protect the project and its contributors.

Enforcement decisions should be proportionate, consistent, and focused on restoring a useful working environment.

## Scope

This code of conduct applies to xMocha project spaces, including GitHub issues, pull requests, discussions, documentation, examples, and other official project channels.
